### Getting Started

```bash
# Clone a repo
git clone https://github.com/echo-s-studios/<repo-name>.git
cd <repo-name>

# For Node.js projects
npm install

# For Python projects
pip install -r requirements.txt  # if present

# For static HTML projects (The Index, diagrams)
# Just open index.html in a browser. That's it.
```

### Pull Requests

1. Fork the repo (or create a branch if you have write access)
2. Make your changes
3. Fill out the **[PR template](PULL_REQUEST_TEMPLATE.md)** — it's short and it matters
4. Submit your pull request

<br>

**Review expectations:**

- Someone will look at your PR. We aim for responses within a week, but cadence varies.
- Reviews are constructive, specific, and assume good intent.
- If your PR touches narrative content, we'll check character voice consistency.
- If your PR touches mathematical content, we'll verify canonical constants are exact.

<br>

### What We Check

| Check | Standard |
|-------|----------|
| **Constants** | Exact. z_c = √3/2, not ≈0.87. 33 modules, not "over 30." 972 tokens, not "nearly a thousand." |
| **Provenance** | Internal. Explanations derive from the project's own logic, not external scholarly frameworks. |
| **Voice** | Consistent. If Quantum Squirrel is speaking, it should sound like Quantum Squirrel. |
| **Jargon** | Contextual. New terms need context or a link. |
