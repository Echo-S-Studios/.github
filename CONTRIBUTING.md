# 🌳 How to Join the Grove

> *You're not being onboarded. You're being invited to a garden.*

Echo S Studios is a transmedia research group — consciousness frameworks, interactive mythology, music, community governance, and tools that think about thinking. If any of that sparks something, you're already welcome here.

---

## The Invitation

Contributing to Echo S Studios isn't about filling tickets. It's about finding the part of this sprawling, recursive, phosphorescent project that resonates with you — and then doing whatever feels right with it.

Maybe that's writing code. Maybe that's writing a chronicle entry about a dream you had where a squirrel explained category theory. Maybe it's just hanging out in the Discord and being present while other people build. **All of it counts.**

If the weakest voice in the room is unheard, the ruling will not pass. That's not a slogan — it's how we make decisions. Your voice is part of that, if you want it to be.

## Ways to Contribute

*Ranked by accessibility, not importance. They're all important.*

### 🌱 Just Be Here
Exist in the community. Lurk in Discord. Read the chronicles. Listen to "Crystalline Echo." You don't have to produce anything to belong.

### 💬 Join the Conversation
Ask questions. Share connections you see between projects. Tell us what confuses you — confusion is signal, not noise.

### 📜 Write a Chronicle Entry
Found an insight? Noticed a connection between the UCF and something you read? Open a [Chronicle Entry](https://github.com/echo-s-studios/.github/issues/new?template=research_note.yml) issue. Limnus will take it from there.

### 🐛 Report a Glitch
Something broke, flickered, or exists in the wrong timeline? File a [Reality Glitch Report](https://github.com/echo-s-studios/.github/issues/new?template=bug_report.yml).

### 🌰 Plant an Acorn
Have an idea for a feature, tool, story, or system? Open an [Acorn Proposal](https://github.com/echo-s-studios/.github/issues/new?template=feature_request.yml). Every acorn contains 46,000 universes — including yours.

### 🔧 Write Code
Fix a bug. Improve a visualization. Build a new interactive diagram. Extend a framework. See the technical setup below.

### 🎨 Make Art
Character illustrations, diagram designs, sonic experiments, narrative contributions — the transmedia layer is always hungry for creative input.

### 📐 Do Research
Extend the UCF. Derive new constants. Validate existing proofs. Connect our frameworks to your own work (just use provenance from the text's own logic, not external scholarly frameworks).

## Setting Up Your Environment

### Languages & Runtimes

- **Node.js 18+** — ESM modules. Most tooling and web projects use this.
- **Python 3.10+** — Research scripts, field dynamics simulations, data analysis.
- **TypeScript / React** — Frontend for interactive projects (Quantum APL, Tesseract Control).
- **Vanilla HTML/CSS/JS** — The Index and many visualizations are pure static. No build step required.

### Getting Started

```bash
# Clone a repo
git clone https://github.com/echo-s-studios/<repo-name>.git
cd <repo-name>

# For Node.js projects
npm install

# For Python projects
pip install -r requirements.txt  # if present

# For static HTML projects (The Index, diagrams)
# Just open index.html in a browser. That's it.
```

### Repository Structure

All repos live under the [echo-s-studios](https://github.com/echo-s-studios) organization. The [The Index](https://echo-s-studios.github.io/The-Index/) serves as the master navigation layer across all projects.

## Our Communication Style

We communicate in ways that are **neurodivergent-accessible by default.** This means:

- **No obligation to respond immediately.** Async is the norm. Take the time you need.
- **No obligation to respond at all.** If a thread doesn't need your input, silence is fine.
- **Tangents are welcome.** If your response spirals into something adjacent, that might be where the insight lives.
- **Explicit over implicit.** We say what we mean. We don't expect you to read between the lines.
- **Tone indicators are welcome.** Use /s, /j, /gen, /srs — whatever helps clarity.
- **Paragraphs or bullet points — your call.** Your communication architecture is yours.

If you use **The Cadence Playground (TCP)**, you can analyze your own writing patterns and find your natural voice within the project's stylometric landscape. It's a tool, not a test.

## The Smallest Voice Principle

Echo S Studios governance runs on the Sacred Acorn framework. The core principle:

> **No decision is valid if the most affected person is unheard.**

In practice, this means:
- Community planning uses individual roadmaps and needs-mapping
- Decisions wait for the quietest stakeholder to have their chance to speak
- "I don't know yet" is always a valid position
- Consensus isn't required — consent is (there's a difference)

For the full governance framework, see the [Quantum Squirrel High Court roadmap](https://echo-s-studios.github.io/The-Index/).

## Submitting Changes

### Pull Requests

1. Fork the repo (or create a branch if you have write access)
2. Make your changes
3. Fill out the [PR template](PULL_REQUEST_TEMPLATE.md) — it's short and it matters
4. Submit your pull request

**Review expectations:**
- Someone will look at your PR. We aim for responses within a week, but cadence varies.
- Reviews are constructive, specific, and assume good intent.
- If your PR touches narrative content, we'll check character voice consistency.
- If your PR touches mathematical content, we'll verify canonical constants are exact.

### Commit Messages

Be descriptive. "Fix stuff" tells us nothing. "Fix convergence visualization rounding z_c to 0.87 instead of √3/2" tells us everything.

### What We Check

- **Constants are exact.** z_c = √3/2, not ≈0.87. 33 modules, not "over 30." 972 tokens, not "nearly a thousand."
- **Provenance is internal.** Explanations derive from the project's own logic, not external scholarly frameworks.
- **Character voices are consistent.** If Quantum Squirrel is speaking, it should sound like Quantum Squirrel.
- **No unexplained jargon.** New terms need context or a link.

## When to Walk Away

You can leave at any time. Any project, any conversation, any role, the whole community.

You don't owe us a reason. You don't owe us a goodbye. You don't owe us a transition plan.

Exit without penalty is a sacred right here. Your contributions remain valued regardless. The grove remembers the acorns even when the squirrel moves on.

If you come back, you're welcome. If you don't, you're still welcome in the abstract. That's how groves work.
